# IBM Cloud vs PolyON Console — UI/UX 메뉴 구조 비교 보고서
**작성자:** Jupiter (팀장) | **일자:** 2026-03-06

---

## 1. 전체 구조 비교

### 1-1. 레이아웃 프레임

| 요소 | IBM Cloud | PolyON Console | 평가 |
|------|-----------|----------------|------|
| **디자인 시스템** | Carbon Design (원조) | Carbon Design (@carbon/react) | ✅ 동일 |
| **상단 헤더** | 로고 + 검색 + 카탈로그 + 관리 + 알림 + 아바타 | 로고 + 도메인명 + 로그아웃 | ⚠️ 빈약 |
| **좌측 네비게이션** | Rail (아이콘 + 호버 확장) | SideNav (열림/닫힘 토글) | ⚠️ 방식 차이 |
| **메인 콘텐츠** | 위젯 기반 대시보드 (편집 가능) | 고정 카드 레이아웃 | ⚠️ 유연성 부족 |
| **우측 패널** | 대시보드 설정 (위젯 추가/범위) | 없음 | ❌ 미구현 |

### 1-2. 상단 헤더 (Global Header)

**IBM Cloud:**
```
[☰] [IBM Cloud] [────── 검색 ──────] [카탈로그] [관리▾]   [?] [셸] [피드백] [추정기] [🔔] [아바타▾]
                                                            └─ 계정 선택기: "3179720 - 화성 최's Account"
```

**PolyON Console:**
```
[☰] [PolyON Console]  [cmars.com]                                              [admin ▾ 로그아웃]
```

**GAP 분석:**

| IBM Cloud 헤더 기능 | PolyON 대응 | 우선순위 |
|---------------------|-------------|----------|
| 글로벌 검색 (자원/제품) | ❌ 없음 | 🔴 높음 — ⌘K 검색 필요 |
| 카탈로그 (서비스 목록) | ❌ 없음 | 🟡 중간 — 앱 카탈로그 |
| 관리 메뉴 (IAM, 결제, 계정) | ❌ 없음 — Settings에 분산 | 🟡 중간 |
| 알림 벨 아이콘 | ❌ 없음 | 🔴 높음 |
| 계정/조직 선택기 | ✅ 도메인명 표시 (읽기전용) | 🟢 |
| 아바타 + 프로필 메뉴 | ⚠️ 로그아웃만 | 🟡 중간 |
| Cloud Shell | ❌ 없음 | 🟢 낮음 |

## 2. 좌측 네비게이션 구조 비교

### 2-1. IBM Cloud Left Navigation

IBM Cloud는 **Rail 패턴** (접힌 상태: 아이콘만 / 호버: 라벨 표시):

```
📊 대시보드
📁 Projects
📋 리소스 목록
────────────
🏢 SAP
────────────
📦 Containers
🗄️ Databases
🖥️ 인프라
📡 관찰 가능성
🤖 Platform Automation
🔒 Security
────────────
[+] 추가
```

**특징:**
- **7개 카테고리** (구분선으로 그룹핑)
- 최상위 레벨만 노출 (서브메뉴는 메인 콘텐츠에서)
- Rail 아이콘은 접혀도 항상 보임
- `+` 버튼으로 카테고리 커스터마이즈

### 2-2. PolyON Console Left Navigation

PolyON은 **SideNav 패턴** (토글 열림/닫힘):

```
📊 대시보드
📱 앱 관리 ▾
    앱 목록 / SSO 설정 / Homepage Sites / Site 생성 / 빌드 현황
👥 디렉터리 ▾
    사용자 관리 / 그룹 관리 / OUs / Computers / Drive 현황
🌳 AD 구조
📧 메일 ▾
    개요 / [디렉터리] 계정·그룹·리스트·도메인·역할·테넌트·API키·OAuth
    [큐&리포트] [이력] [로그&추적] [스팸관리] [트러블슈팅] [관리]
💬 채팅 ▾
    개요 / 팀 / 채널 / 사용자 / 설정
🤖 AI ▾
    대시보드 / Models / Agents / Memory / API Keys / Usage / Settings
🔄 자동화 ▾
    개요 / 워크플로우 / 실행 이력 / 설정
⚙️ BPMN ▾
    프로세스 / 인스턴스 / 태스크 / 히스토리 / 인시던트 / 배포 / DMN
🌐 네트워킹 ▾
    DNS / DC / VPN / Firewall
📦 컨테이너 ▾
    개요 / 토폴로지 / 리소스
🗄️ 데이터베이스 ▾
    PostgreSQL / Redis / Elasticsearch
📈 모니터링 ▾
    대시보드 / Alerts / Sentinel / Logs
🔒 보안 ▾
    Audit Log / Policies / GPO / ACL / 접근 정책 / 감사 로그
⚙️ 설정 ▾
    일반 / 도메인 / TLS / 백업 / 시스템 정보 / 인증 / 계정 / SMTP / 초기화 / 이력 / 미러링 / Workstream
```

**특징:**
- **15개 카테고리** — IBM Cloud의 2배 이상
- 서브메뉴가 SideNav 안에서 모두 펼쳐짐 (특히 메일: 30개+ 항목)
- 스크롤이 길어질 수밖에 없는 구조

## 3. 핵심 차이점 분석

### 3-1. 네비게이션 패턴

| 항목 | IBM Cloud | PolyON | 판정 |
|------|-----------|--------|------|
| **패턴** | Rail (항상 보이는 아이콘 + 호버 확장) | SideNav (토글 열림/닫힘) | IBM 방식이 공간 효율적 |
| **깊이** | 1단계 (카테고리만) | 2~3단계 (카테고리 > 서브메뉴 > 그룹) | PolyON 과도하게 깊음 |
| **항목 수** | ~10개 (최상위) | ~15개 카테고리, 총 100개+ 항목 | PolyON 과밀 |
| **서브 네비게이션** | 메인 콘텐츠 영역에서 탭/좌측 패널 | 사이드 네비에서 모두 처리 | IBM 방식이 깔끔 |
| **구분선** | 카테고리 그룹 구분 | 없음 | PolyON에 구분선 필요 |

### 3-2. 대시보드 패턴

| 항목 | IBM Cloud | PolyON | 판정 |
|------|-----------|--------|------|
| **위젯 시스템** | ✅ 추가/제거/편집 가능 | ❌ 고정 카드 | IBM 방식 도입 필요 |
| **대시보드 선택기** | ✅ 여러 대시보드 전환 | ❌ 단일 | 장기 목표 |
| **"리소스 작성" CTA** | ✅ 항상 보이는 주요 액션 | ❌ 없음 | 도입 권장 |
| **참고(Notes) 위젯** | ✅ Markdown 편집 | ❌ 없음 | 조직 메모용 |
| **빌드(튜토리얼) 카드** | ✅ 온보딩 가이드 | ❌ 없음 | 첫 사용자 UX |

### 3-3. 정보 밀도

| 항목 | IBM Cloud | PolyON |
|------|-----------|--------|
| **메일 서브메뉴** | 해당 없음 | 7개 그룹, 25개+ 항목 → **별도 페이지 내 탭으로 분리 필요** |
| **BPMN 서브메뉴** | 해당 없음 | 8개 항목 → 적정 |
| **설정 서브메뉴** | 관리 > 계정/IAM/결제 분리 | 12개 항목 한 그룹 → **분리 필요** |

## 4. IBM Cloud에서 배워야 할 핵심 패턴

### 4-1. 🔴 즉시 적용 권장

1. **Rail Navigation 전환**
   - 현재 SideNav → Carbon `UIShell` Rail 패턴으로 전환
   - 접힌 상태: 아이콘만 보이는 얇은 Rail (48px)
   - 호버/클릭: 라벨 + 서브메뉴 표시
   - 메인 콘텐츠 공간 확보

2. **서브 네비게이션 분리**
   - 메일, 설정 등 항목이 많은 카테고리: **메인 콘텐츠 내 좌측 서브 네비게이션**
   - IBM Cloud의 리소스 상세 페이지처럼 `Content > [Left Sub-Nav | Main Content]`

3. **글로벌 검색 (⌘K)**
   - 상단 헤더에 검색 바 추가
   - 사용자, 그룹, 서비스, 설정 통합 검색

4. **알림 시스템**
   - 헤더에 🔔 알림 아이콘
   - 시스템 이벤트, 서비스 상태 변경, 보안 알림

### 4-2. 🟡 단기 적용 (1-2주)

5. **카테고리 그룹핑 + 구분선**
   ```
   [기본]     대시보드
   ─────────────────
   [ID/접근]  디렉터리 / 보안
   ─────────────────
   [서비스]   메일 / 채팅 / AI / 자동화 / BPMN
   ─────────────────
   [인프라]   네트워킹 / 컨테이너 / 데이터베이스
   ─────────────────
   [운영]     모니터링 / 설정
   ```

6. **대시보드 위젯 시스템**
   - 서비스 상태, 사용자 수, 스토리지, 메일 현황 등 위젯화
   - 관리자가 위젯 추가/제거/재배치

7. **프로필 메뉴 강화**
   - 아바타 + 이름 + 역할
   - 내 계정, 보안 설정, 로그아웃

### 4-3. 🟢 중기 (Workstream 연계)

8. **"작업 만들기" 글로벌 CTA**
   - IBM의 "리소스 작성"에 대응
   - PolyON에서는 "업무 시작" / "사용자 추가" 등 주요 액션

9. **대시보드 커스터마이즈**
   - 여러 대시보드 프리셋 지원
   - 역할별 기본 대시보드 (관리자 vs 일반 사원)

## 5. 메뉴 구조 재편 제안

### AS-IS (현재 15개 평면 카테고리)
```
대시보드 / 앱 / 디렉터리 / AD 구조 / 메일 / 채팅 / AI /
자동화 / BPMN / 네트워킹 / 컨테이너 / DB / 모니터링 / 보안 / 설정
```

### TO-BE (IBM Cloud 스타일 그룹핑, 10개)
```
📊 대시보드
─────────────
👥 ID & 접근
    사용자 / 그룹 / OUs / Computers / AD 구조 / 보안(ACL·GPO·정책)
─────────────
📱 서비스
    메일 / 채팅 / AI / 앱
─────────────
⚙️ 자동화
    워크플로우(n8n) / BPMN(Operaton)
─────────────
🖥️ 인프라
    네트워킹 / 컨테이너 / 데이터베이스 / 스토리지
─────────────
📈 관찰 가능성
    모니터링 / Logs / Alerts / Sentinel
─────────────
⚙️ 관리
    설정 / 도메인 / 백업 / 시스템 정보
```

**변경 포인트:**
- 15개 → 7개 그룹 (구분선 포함)
- "디렉터리 + 보안" → "ID & 접근"으로 통합 (IBM의 Security 패턴)
- "자동화 + BPMN" → "자동화"로 통합
- "모니터링" → "관찰 가능성"으로 확장 (IBM의 Observability 패턴)
- "설정" → "관리"로 격상

## 6. Look & Feel 비교

| 요소 | IBM Cloud | PolyON | 차이 |
|------|-----------|--------|------|
| **배경색** | `$ui-background` (#f4f4f4) | 흰색 또는 유사 | 비슷 |
| **헤더 색** | 검정 (#161616) | 검정 또는 다크 | ✅ 동일 |
| **카드 스타일** | 흰색 카드 + 그림자 없음 + 테두리 | 유사 | ✅ |
| **아이콘** | Carbon Icons (outline, 20px) | Carbon Icons | ✅ 동일 |
| **폰트** | IBM Plex Sans | IBM Plex Sans | ✅ 동일 |
| **간격** | 8px 기반 Carbon spacing | 8px 기반 | ✅ 동일 |
| **버튼 스타일** | Primary(파랑), Ghost, Tertiary | 유사 | ✅ |
| **위젯 헤더** | `heading-03` + "View all" 링크 | 유사 | ✅ |

**L/F는 이미 높은 수준으로 일치.** 구조와 패턴의 차이가 핵심.

## 7. 결론

**PolyON Console은 Carbon 컴포넌트를 올바르게 사용하고 있지만, IBM Cloud의 UX 패턴(정보 구조, 네비게이션 계층, 위젯 시스템)을 아직 충분히 반영하지 못했습니다.**

가장 임팩트 있는 3가지 개선:
1. **Rail Navigation** — 공간 효율 + 깔끔한 카테고리
2. **메뉴 그룹핑 (15→7)** — 정보 과밀 해소
3. **글로벌 검색 + 알림** — 헤더 기능 강화

이 3가지만 적용해도 IBM Cloud 수준의 UX에 근접합니다.
